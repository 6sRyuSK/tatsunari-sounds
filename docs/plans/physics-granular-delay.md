## 0. 要約

入力のトランジェントを検出すると、回転する 2D ボックスの中にボールがスポーンする。
ボールは物理演算(初速・重力・ドラッグ・反発)で運動し、**壁に衝突するたびに、
そのボールを生んだ音のグレインが 1 発再生される**。`Time` × `Count` はディレイタイム/
フィードバックに似た操作感を与えるが、それは**無音のトリガー(ボール)の生成
スケジュール**にすぎない(§6-D13)。発音はすべて衝突の瞬間であり、そのタイミング・
減衰・パンは物理から創発する。ビジュアライザはその物理そのもの。

- 構造は **2 階層**: 1 オンセットが `Time` 間隔で `Count` 本のボールを生む
  「トリガー列」(無音)と、**唯一の発音イベントである「衝突」**。古典的な意味の
  ディレイタイムは厳密には存在せず、発音タイミングが物理由来の揺らぎを帯びるのが
  本機のアイデンティティ(§6-D13。従来の `Density(Hz)` も不在)。
- パラメータの異なるボールを最大 4 スロット(A–D)同時に飛ばせる。
- 音声フィードバックループは持たない: 減衰列はトリガー列(§4.1)、世代シマーは
  Refeed(§4.8)。いずれも構造的に発散不可能。

## 1. コンセプトと位置づけ

### 1.1 エレベーターピッチ

「音を投げ込むと、回る箱の中でコロコロ跳ねて返ってくるディレイ」。
リズムはテンポグリッドではなく重力と幾何が作る。壁にぶつかった瞬間だけ音が出る、
というルールが 1 つあるだけで、accelerando(重力バウンス)、poly-rhythm(多角形)、
うねる周期(偏心回転)がすべて自然に出てくる。

### 1.2 既存 Tatsunular Delay(`granular-delay`)との違い

| | Tatsunular Delay | 本機 |
|---|---|---|
| グレインの発火 | クロック駆動(`Density Hz` + jitter) | **衝突イベント駆動** |
| エコーの減衰 | feedback ゲイン(音声ループ) | トリガー列の減衰(`Count` × `Bounce`)+ 物理 + Refeed 世代 |
| リズムの性格 | 定常的な雲・テクスチャ | トリガー列(無音) × 衝突発音(物理由来の揺らぎ)の 2 階層 |
| パン | ランダム spread | 衝突位置(物理)由来 |

役割が異なるため共存させる(置き換えではない)。グレイン再生の実装語彙
(Hann 窓・rate 読み・プール)は `core/GranularDelay.h` と揃える。

## 2. 用語とオブジェクトモデル

- **World(ワールド)** — 1 つの 2D 空間。**箱は 1 個で全スロット共有**(§6-D2)。
- **Box(箱)** — 回転する正多角形または円。壁がグレインのトリガー面。
- **Slot(スロット)** — ユーザー提案の「ボールインスタンス」。ボールのパラメータ
  一式を持つ。A–D の 4 基、A のみデフォルト有効。
- **Ball(ボール)** — スロットから生まれる実体。生成時に「アンカー」(自分を生んだ
  入力音のリングバッファ上の時刻)を 1 つ持つ。
- **Sequence(シーケンス)** — 1 オンセットが 1 スロットに生むボール(トリガー)の
  列(`Time` 間隔で `Count` 本)。スポーン自体は無音 — 発音は衝突のみ
  (§4.1、§6-D13)。
- **Collision(衝突)** — ボールと壁の接触イベント。1 衝突 = 最大 1 グレイン。
- **Grain(グレイン)** — Hann 窓の短い再生断片。リングバッファのアンカー近傍を読む。
- **Generation(世代)** — Refeed で親ボールから子ボールが生まれた回数(§4.8)。

## 3. シグナルフロー

```
in L/R ──┬───────────────────────────── dry ────────────┐
         ├─ mono sum ─ OnsetDetector ─ spawn events     ├─ Mix ─ Out
         └─ stereo ring buffer (18 s)                   │
                   ↑ read (anchored, per-grain rate)    │
              GrainEngine ── grains ─ Tone LP ── wet ───┘
                   ↑ collision events (sample-accurate)
              PhysicsWorld: Box(shape/size/spin/pivot/gravity)
                            × Balls(slot A–D params)
```

音声ループは存在しない(純フィードフォワード)。Refeed は「音を書き戻す」の
ではなく「減衰した子ボールを生む」(§4.8)。

## 4. 動作仕様

### 4.1 オンセット検出とスポーン

- 検出はモノサム上の 1 系統(グローバル)。`EnvelopeFollower.h`(fast/slow ペア)
  ベースのオンセット検出。
- パラメータ: `Sense`(しきい値 dBFS)、`Retrig`(再トリガー禁止時間)。
- **絶対 floor −80 dBFS**(退行クラス J): floor 以下の入力では絶対にスポーン
  しない。silence テスト必須。
- 1 オンセット = 有効な全スロットに 1 イベント配信。各スロットは**シーケンス**を
  開始する: ボール k(k = 1..`Count`)を
  `t_onset + Pre-Delay + (k−1) × Time` にスポーンする(§6-D1)。
  - `Time` = **トリガー間隔**、`Count` = **トリガー本数**。操作感はディレイタイム/
    フィードバックに似るが、スポーンは無音のトリガー生成にすぎず**発音は衝突のみ**
    (§6-D13)。`Count = 1` なら 1 オンセット 1 ボール。
  - シーケンスの全ボールは**同じアンカー**(そのオンセット)を継承する。
    `Motion = 0` なら全ボールが同じ誕生音を鳴らすため、トリガー列が「同じ音の
    減衰列」を作る — ただし個々の発音時刻は各ボールの衝突が決める(§6-D13。
    `Motion = +100` なら読む音の古さがタップ状に固定される、§4.6)。
  - **トリガー列の減衰(§6-D11)**: ボール k は「生まれつき (k−1) 回バウンド済み」
    として初速 `Speed × Bounce^(k−1)` で生む(推奨案)。衝突速度→ゲイン(§4.5)に
    より音量が自動で幾何減衰し、後のボールほど衝突が遅く疎になる(テープエコー的
    劣化)。低 `Bounce` では後段がエネルギー死で自然消滅 = `Count` は上限、実効の
    本数は `Bounce` が決める。代替は純ゲイン減衰(リズム不変)— taste 判断。
  - 音が鳴るのはスポーンではなく**初回衝突**。発音タイミングは
    `Time + 初回飛行時間` で揺れるが、これは欠点ではなく本機の特徴そのもの
    (§6-D12/D13)。タイトにしたければ `Box Size ≪ Time`。厳密グリッド化は
    拡張(§14)。
  - 新しいオンセットが来てもシーケンスは巻き直さず**重奏**する。生存上限は
    グローバルプール(32)の oldest-steal が担保。
  - 回転する箱では、ランダム 0% でもボール k ごとに箱の位相が異なるため、
    後続のボールほどバウンスパターンが進化する(意図した仕様)。
- スポーン位置: 箱の中心 + `Spawn Spread`(半径 R に対する % の一様円板ジッタ)。
  完全対称な初期条件は退化軌道(同じ 2 点を往復)に落ちるため、デフォルトで
  少量のジッタを入れる。
- 初速の向き: スロットの `Direction`(角度)を `Dir Random` % で一様ランダムに
  ブレンド。デフォルトは 100%(完全ランダム)。

### 4.2 物理ワールド — 座標系・単位・時間スケール

- 単位系: 箱の外接円半径 **R = 1**。物理は秒領域で積分(サンプルレート非依存)。
- **`Box Size` が「テクスチャの時間スケール」を決める**(§6-D3):
  `Box Size = T` 秒 ⇔ 「基準速度のボールが箱の直径 2R を横断するのに T 秒かかる」。
  すなわち基準速度 `vRef = 2R / T`。ボールの実速度 = `Speed × vRef`。
  → `Time`(トリガー間隔、§4.1)が列の周期を、`Box Size` / `Speed` が各ボールの
  衝突リズム(グラニュラーの目の細かさ)を決める。発音時刻そのものは常に衝突が
  決める(§6-D13)。
- `Box Size` はテンポシンク可(音価 = 横断時間)。
- ボール半径は R 比 %。`min(有効半径) ≤ 0.4 × 内接円半径` にクランプ
  (箱よりデカいボール対策)。

### 4.3 回転する箱 — 移動壁とエネルギー、クランプ

- `Spin`: 双極性(±方向)、free では rev/s、sync では「1 回転あたりの音価」。
  デフォルトは 0 ではなく**ゆっくり回っている**(ユーザー提案の「デフォルトで
  四角のボックスが回っている」を反映。初期値は taste — §15)。
- `Pivot X/Y`: 回転軸の位置。(0,0)=中心。±1 × R の範囲。軸をずらすと箱が
  偏心して振れ、衝突タイミングが周期的にうねる(ワウ的なグルーヴ)。
- 衝突は**移動壁**として解く: 接触点の壁速度 `v_wall`(回転による接線速度)を
  使い、相対速度の法線成分を反発させる
  `v' = v − (1 + e) · ((v − v_wall)·n) n`。
  → **回転はボールにエネルギーを注入できる**(タンブラーが中身をかき回す)。
  これは意図した仕様(回転を上げるとエコーが死なずに暴れ続ける)。
- 安全クランプ(必須): ボール速度上限 `|v| ≤ 8 × vRef`。グレインゲインは
  §4.5 のマッピングで **1.0 で飽和**。→ 長時間ホールドでも出力は有界
  (退行クラス C の「現実的ピーク上限」テスト対象)。
- `Ball Collide` トグル(デフォルト off): ボール同士の弾性衝突(質量 ∝ r²)。
  on でも**ボール同士の衝突はグレインを発火しない**(v1。拡張 §14)。
  壁衝突のみが `Life Bounces` にカウントされる。

### 4.4 ボールの運動 — 「加速度(減衰度)カーブ」の分解

ユーザー提案の「初速、加速度(減衰度)カーブ」を次の 4 つに分解する(§6-D4):

| 要素 | パラメータ | 物理 | 音への効き |
|---|---|---|---|
| 初速 | `Speed`(スロット) | スポーン時の速さ | エコー密度の基準 |
| 定加速度 | `Gravity`(グローバル) | 一定の下向き加速度 | バウンスの accelerando。0 = ビリヤード(等速) |
| 経時減衰 | `Drag`(スロット) | 速度比例の空気抵抗 | エコー間隔が徐々に開く(rit.) |
| 減衰の形 | `Decay Curve`(スロット) | Drag 係数をボール年齢で変化させるプロファイル(前重心 ↔ 線形 ↔ 後重心) | 「最初に一気に落ちる」↔「最後に急に死ぬ」 |
| 衝突減衰 | `Bounce`(スロット、反発係数 e) | 衝突ごとのエネルギー保持率 | 1 衝突ごとの音量減衰 = feedback 相当 |

- 重力は**ワールド(画面下)固定**で箱と一緒に回らない。回る箱 × 固定重力 =
  タンブラーの気持ちよさの源泉。
- e(`Bounce`)の上限は **100%**(>100% の増幅反発は禁止 — 退行クラス A 域。
  拡張候補 §14)。
- 静止判定: 法線衝突速度 `< 0.02 × vRef` の接触はグレインを発火せず滑らせる
  (転がり接触のマシンガン化防止)。加えてボールごとにグレイン発火の最小間隔
  15 ms を内部固定で持つ。

### 4.5 衝突 → グレイン変換

- **タイミング**: 衝突時刻はサンプル精度でグレイン開始に反映(§4.10)。
- **ゲイン**: `g = min(1, |v_impact·n| / vRef)`。反発 e のボールは衝突ごとに
  インパクト速度が e 倍 → グレインゲインも e 倍(検証しやすい独立オラクル、
  §10-T2)。上限 1.0 で飽和(壁で音が増えることはない)。
- **パン**(`Pan Mode`、スロット):
  - `Physics`(デフォルト): 接触点の x 座標(ワールド系)→ 等パワーパン。
    箱の左壁に当たれば左から鳴る。ビジュアライザと音像が一致する。
  - `Center` / `Random`: 従来型。
- **グレイン内容**: リングバッファをアンカー位置(§4.6)から `Grain` 長ぶん、
  Hann 窓、per-grain 再生 rate = `2^((Pitch + PitchRand·rand)/12)`。
  `Reverse` % の確率で逆再生。実装語彙は `GranularDelay.h` の Grain 構造を踏襲
  (プール固定・rate 読み・d クランプ)。
- グレインプールはグローバル 64(core 精例 `kMaxGrains` と同値)。満杯時は
  新規をドロップ(core と同じポリシー)。

### 4.6 グレインソースのアドレッシング — Source クラスタ(§6-D5)

**「衝突時に何の音を鳴らすか」は本機のグラニュラーとしての性格を決める心臓部**
(ユーザー指摘 2026-07-06)。単一ノブ(旧 `Warp`)では表現の幅が足りないため、
読み出し位置を「基準点 + 移動 + 乱れ」に直交分解した **Source クラスタ
(3 パラメータ)** に拡張する。

ボールは生成時に「自分を生んだオンセットのリングバッファ時刻」= アンカー
`anchor0` を記憶する。そのボールの n 回目のグレイン発火(衝突)での読み出し:

```
head(t_c) = anchor0 + Motion × (t_c − t_spawn) + Step × n + Spray × rand()
```

読んでいる音の「古さ」は
`τ = PreDelay + (1 − Motion) × (t_c − t_spawn) − Step × n` と等価。

| パラメータ | 範囲 | 役割 |
|---|---|---|
| `Motion`(旧 Warp を双極化) | −100〜+100 % | 読みヘッドの実時間速度。**0 % = 誕生トランジェントに固定**(ボール = 音の化身)、**+100 % = 入力追従**(τ 一定 = 古典ディレイ)、**負 = 過去へ遡行**(生きるほど古い音へ潜る) |
| `Step` | −500〜+500 ms | **衝突 1 回ごと**に読みヘッドを一定量シフト。+ でフレーズを 1 かけらずつ読み進める「フレーズスキャナ」(重力アッチェレランドと組むと読み出しも加速)。− で逆走 |
| `Spray` | 0〜500 ms | グレインごとの読み位置ジッタ(±一様)。古典グラニュラーの雲状スメア。**タイミングの乱れ(物理)と位置の乱れ(Spray)は別軸** — v0 ドラフトで Position Jitter を削ったのは誤りで、ここで復活 |

代表的なレシピ(全部このクラスタだけで出せる):

| 狙い | Motion | Step | Spray |
|---|---|---|---|
| ボール = 音の化身(リズミックな反復) | 0 | 0 | 0 |
| 古典的グラニュラーディレイ(入力追従の雲) | +100 | 0 | 30–80 ms |
| タップ状の内容遅れ(シーケンス §4.1 と併用: ボール k の**読む音の古さ**が τ = PreDelay+(k−1)×Time に固定される。発音時刻は衝突依存のまま — D13) | +100 | 0 | 0 |
| フレーズスキャナ(跳ねるたびに続きを読む) | 0 | +50〜+150 | 0–20 ms |
| 記憶ダイブ(跳ねるたびに古い音へ) | −50〜−100 | または負の Step | 少量 |
| 霧/クラウド(定位置のスメア) | 0 | 0 | 200 ms+ |

規則(クランプと寿命):

- `head` は書き込みヘッドを追い越せない(`τ ≥ 1 sample` にクランプ —
  `GranularDelay.h` の d クランプと同じ規約)。
- **ホライズン死は「アンカー年齢」ではなく実効 τ ベースに再定義**: τ が 34 s
  (§4.9)に達したボールは retire。負 `Motion` / 大きな `Step` の遡行はリングを
  伸ばさず retire で打ち切る(最古領域をループ再生する artifact を出さないため)。
- `Motion` / `Step` はボールごとの決定的な読みヘッド軌跡(1 kHz ティック評価)、
  `Spray` のみグレインごとの乱数(決定的シード)。n は**グレインを発火した衝突**
  のみ数える(§4.4 の微小衝突ガードで抑制された接触は含まない)。
- シーケンス(§4.1)の全ボールは同じ `anchor0` を継承する。

### 4.7 寿命と消滅

- `Life Mode` トグル(ユーザー提案どおり):
  - `Time`: 0.1–16 s。経過で消滅。
  - `Bounces`: 1–99 回。**壁反射のみ**カウント。
- 消滅時: 進行中のグレインは最後まで再生(新規発火だけ止める)。ビジュアルは
  フェードアウト。クリックは構造上出ない(グレインは常に窓済み)。
- 暗黙の消滅(モードに関係なく常時):
  1. エネルギー死: `|v| < 0.01 × vRef` が 250 ms 続いたら消滅(ゾンビ防止)。
  2. ホライズン死: 実効読み位置の古さ τ(§4.6)がリングの保持限界を超えたら
     消滅(§4.9)。

### 4.8 Refeed — 世代フィードバック(音声ループを持たない feedback)

フィードバック相当の減衰列はトリガー列(§4.1)が担うため、Refeed は**世代シマー/
テール延長**という直交した役割を持つ(§6-D6)。いずれも**音声ループ方式は採らない**:
衝突駆動ではグレインの重なりが不定で、N 個のボールが同一アンカーを同時に読むと
ループゲインが `feedback × N` になり得る(退行クラス A の設計違反)。代わりに:

- `Refeed`(グローバル、0–95%): ボールが**消滅するとき**、確率/ゲイン `Refeed`
  で**子ボール**を最終衝突位置からスポーンする。
- 子は親のパラメータ一式を継承しつつ:
  - `gainScale(child) = gainScale(parent) × Refeed` — 世代ごとに幾何級数で減衰。
  - `Pitch` はスロット値を**累積**(gen 2 = 2×Pitch semitones …)。
    → 音声ループなしでシマー的なピッチ螺旋が得られる。
  - アンカーは親のものを継承(同じ音の残響世代)。
- 世代上限 8。`gainScale < −60 dB` でも打ち切り。
- **構造的に発散不可能**(純フィードフォワード、各世代ゲイン ≤ Refeed^gen < 1)。
  クラス A のワーストケーステストは「それでも」全レートで回す(§10-T3)。

古典的な音声ループ Regen(ウェットをリングに書き戻す)は v1 非搭載。搭載する
場合はループゲイン正規化の不変量設計が必要(拡張 §14、人間相談案件)。

### 4.9 メモリホライズン(§6-D7)

`Motion < +100%` のグレインは過去のアンカーを読み続けるため、リング長が寿命を
規定する:

- 最悪アンカー年齢 = max Pre-Delay 1 s + (max Count − 1) × max Time 2 s = 14 s +
  max Life Time 16 s + max Grain × rate マージン ≈ 2.5 s → リングバッファ =
  **34 s**。ステレオ。`prepareToPlay` で最悪ケース確保(退行クラス D)。
  テンポシンクで `Time` が 2 s を超える場合は 2 s に実効クランプ
  (明示クランプ+テスト)。
- メモリ見積: 48 kHz float ≈ 13.1 MB、192 kHz float ≈ 52.2 MB
  (double だと @192k で 100 MB 超になるため **float 格納を仕様とする**)。
- 実効 τ(§4.6)が 34 s を超える状況(Bounces モードの超スローボール、負 `Motion`
  や `Step` での遡行等)ではボールをホライズン死させる(クランプで誤った音を読ませない — クラス D の
  「明示クランプ+テスト」)。

### 4.10 決定性とサンプルレート非依存(§6-D8)

- 物理は**固定 1 kHz ティック**(秒領域、絶対時刻アンカー)で積分し、衝突を
  検出したティック内で**二分法(上限 8 反復)でサンプル精度に精密化**する。
  → タイミングはサンプルレートにもブロックサイズにも依存しない。
- 乱数は core 精例の xorshift。シードはボールごとに決定的
  (`hash(slotIndex, spawnCounter)`)。→ 同一入力・同一パラメータなら
  オフラインレンダーは**ビット再現**(§10-T6 で 64 vs 512 サンプルブロックの
  一致を検証)。
- 1 ティック内の多重衝突は TOI 順に逐次解決、ボールあたり上限 4 回/ティック
  (RT 上限)。ボールが壁にめり込む遷移(箱の形状/サイズ/軸の変更中)は法線方向
  push-out で解消し、速度は増やさない(爆発防止)。

### 5.4 平滑化と適用タイミング(退行クラス F)

- 毎サンプル `SmoothedValue`: `mix` `outGain` `tone` `Gain`(音声経路の連続量)。
- 物理パラメータ(`spin` `boxSize` `pivot` `gravity` 等)は 1 kHz ティックで
  スムージング適用(イベント列にしか影響しないためジッパーは構造上出ない。
  `boxSize` 急変は §4.10 の push-out で安全)。
- グレイン単位でラッチ: `Pitch` `Grain` `Reverse` `Spray`(発火時に確定、
  再生中は不変 — クリックなし)。`Motion` / `Step` はボールの読みヘッド軌跡
  として 1 kHz ティックで評価(§4.6)。

## 6. 明言されていなかった点への提言(決定リスト)

実装前に確定が必要な解釈。**太字が本書の推奨**(異議がなければこれで確定)。

| # | 論点 | 提案 |
|---|---|---|
| D1 | 「ボールの数」の意味 | **確定(ユーザー裁定 2026-07-06)**: `Count` = 1 オンセットが `Time` 間隔で生む**ボール(トリガー)本数**、`Time` = 2 個目以降のボールが出現する間隔(per-slot 新設)。操作感はフィードバック/ディレイタイムに似るが**等価ではない — スポーンは無音で、発音は衝突のみ(D13)**。旧案(最大同時生存数)は破棄し、生存上限はプール oldest-steal が担保。`Time` 最小 10 ms がバースト射出を実質的に兼ねる |
| D2 | 箱は 1 個か、スロットごとか | **共有ワールド 1 箱**(ビジュアルの明快さ、「同じ部屋で違うボールが跳ねる」)。スロット別の箱(マルチチェンバー)は拡張 |
| D3 | 時間スケールの定義 | 時間の入口は 2 つ: `Time`(トリガー間隔)と `Box Size`(基準横断時間 = 衝突テクスチャの速度、§4.2)。**どちらも発音時刻を直接は決めない**(D13)。両方テンポシンク可。HUD にトリガー列と推定初回飛行時間を表示 |
| D4 | 「加速度(減衰度)カーブ」の解釈 | **Gravity(グローバル定加速度)/ Drag(経時減衰)/ Decay Curve(減衰プロファイル)/ Bounce(衝突減衰)の 4 要素に分解**(§4.4)。「1 本のカーブ」に畳みたい場合は要相談 |
| D5 | 衝突時に「何の音」を再生するか | **グラニュラーの性格を決める心臓部として単一ノブ(旧 Warp)をやめ、Source クラスタ(`Motion` 双極 / `Step` / `Spray`)に拡張**(§4.6、ユーザー指摘 2026-07-06)。誕生音固定・入力追従・遡行・フレーズスキャン・雲スメアを同一の 3 ノブで連続可変。**`Motion` のデフォルト(0 = 化身 / +100 = 古典)は最重要 taste — §15** |
| D6 | フィードバックの構造 | フィードバック相当の減衰列 = **トリガー列(D1、D13 の注意つき)**。`Refeed` 世代スポーン(§4.8)は直交するシマー/テール延長。どちらも音声ループなしで構造的に発散不可能。古典 Regen(書き戻し)は拡張(不変量設計が必要) |
| D7 | 寿命・メモリの上限(未明言) | **リング 34 s のホライズン**を仕様化(§4.9、トリガー列 `(Count−1)×Time` を含む)。Life Time 上限 16 s・`Time` 実効上限 2 s、超えるボールはホライズン死 |
| D8 | 決定性(未明言) | **同一入力→ビット再現**を仕様に含める(固定 1 kHz ティック+サンプル精度精密化+決定的シード、§4.10)。オートメーション/オフラインレンダーの再現性とテスト容易性のため |
| D9 | 検出系(未明言) | グローバル 1 系統のオンセット検出(`Sense`/`Retrig` + 絶対 floor −80 dB)。スロット別検出はしない |
| D10 | 静止・転がり時の挙動(未明言) | 微小衝突はグレイン発火しない(§4.4)+ボールごと発火最小間隔 15 ms(マシンガン防止、内部固定) |
| D11 | トリガー列の減衰則 | **初速減衰(推奨)**: ボール k の初速 = `Speed × Bounce^(k−1)`(「生まれつきバウンド済み」— 後のボールほど静かで遅く疎、テープ的劣化。低 Bounce では後段が自然消滅し、実効の本数を Bounce が決める)。代替: 純ゲイン減衰(全ボール同リズム、音量のみ幾何減衰)。**taste 判断 — §15** |

## 7. UI 仕様(factory_ui 準拠)

- レイアウト:
  - ヘッダ: FactoryChrome + `PresetSelectorController`(標準)。

    ジェスチャ通知)。HUD: トリガー列(`Time` × `Count`)・推定初回飛行時間 /
    スロット別生存数。
  - 右カラム: World(Shape/Size/Spin/Pivot/Gravity/Ball Collide)、
    Detect(Sense/Retrig/Spread)、Out(Refeed/Tone/Mix/Output)。
  - 下段: スロットタブ A–D(有効 LED + 選択)+ 選択スロットのノブバンク
  (Physics / Source(`Motion` `Step` `Spray`) / Sound の 3 グループで区分け)。

  スキルで確認。パレットに 4 アクセントが無ければ**人間相談** — パレットの
  fork は禁止されているため)。
- 数値表示は共有ヘルパ(`setSliderDecimals` 等)で桁固定(退行クラス K)。
- **スレッド契約**(退行クラス I): 物理は**オーディオスレッドの DSP**(タイミング
  が音楽的本体のため。UI スレッド物理は不可)。UI へは
  - イベント SPSC FIFO(spawn/hit/die、有界・上書き)
  - 60 Hz 位置スナップショット(atomic インデックスのダブルバッファ)
  で一方向ミラーする。エディタ閉時は書き捨て。UI→DSP は APVTS 経由のみ。

## 9. RT 安全と退行防止ゲート(クラス対応)

`processBlock` 系は allocation/lock/syscall なし(プール・リング・FIFO は
`prepareToPlay` で最悪ケース確保)。pluginval strictness 5 対象。

| クラス | 本機での対応 |
|---|---|
| A 発散 | 音声ループ自体を持たない(Refeed は世代ゲイン ≤ Refeed^gen)。それでも最悪ケース(最大 Spin/Bounce/Refeed/32 балls)でインパルス応答エネルギー非増加を全 6 レート検証 |
| B 共振上限 | 共振フィルタなし(Tone は 1-pole)。該当ゲートなしを明記 |
| C finite | グレイン合算→出力ノードに finite ガード(非 finite でグレイン kill + リングflush、自己回復)。長時間ホールドで `allFinite` + **peak < 4.0**(現実的上限) |
| D バッファ | リング 34 s を最悪ケース(PreDelay + (Count−1)×Time + Life + グレインマージン)から導出(§4.9)。ホライズンと Time シンクは明示クランプ(ボール retire)+テスト |
| E リセット | `prepareToPlay` 末尾で `reset()`(全ボール kill・リング/検出器/RNG 初期化)。bypass 解除・channel 遷移でも同様 |
| F 平滑化 | §5.4 のとおり(SmoothedValue / ティック平滑 / グレインラッチ) |
| G レート行列 | 全テスト `sampleRatesFromArgs()`。テスト内 FFT は `fftOrderForSampleRate` |
| H バグ経路 | ワースト設定(最大 Spin・e=1・Drag 0・全ボール)を実際に踏む。オラクルは §10(解析解) |
| I atomic | ビジュアライザ共有は SPSC FIFO + atomic ダブルバッファのみ(§7) |
| J 絶対 floor | 検出器 −80 dBFS floor + silence テスト(スポーン 0・ウェット 0) |
| K GUI | factory_ui 共有ヘルパで桁・幅・描画空間を統一 |
| L コム | dry と wet は「別時刻の音」の混合であり並列同相合成ではない(クラス L 非該当を明記)。equal-gain mix |
| M 線形位相 | FIR クロスオーバー不使用(非該当) |

| T4 | エネルギー注入の有界性 | 最大 Spin・e=100%・Drag 0・32 ボール・60 s ホールド → `allFinite` + peak < 4.0 + ボール速度 ≤ 8 vRef(クランプ検証、クラス C/H) |

| T6 | 決定性 | 同一入力をブロック 64 と 512 で処理 → ウェットがビット一致(ティック絶対時刻・シード・TOI 精密化の検証) |
| T7 | リセット/遷移 | prepare→process→prepare で残留なし、bypass 遷移でボール全滅(クラス E) |
| T8 | ホライズン | アンカー年齢 33.9 s のグレインが正音、34 s 超でボール retire。Time シンクの 2 s 実効クランプも検証(クラス D の明示クランプ) |
| T9 | パン則 | Physics パン: 既知の接触 x → 等パワー則の L/R ゲインと一致(閉形式) |
| T10 | Life Bounces | e=1・ビリヤード設定で正確に N 回のグレイン後に停止(T1 の時刻列と併用) |
| T11 | トリガー列 | スポーン時刻は閉形式 `t_onset + PreDelay + (k−1)×Time`(スポーンは無音 — 出力ゼロを assert)。ビリヤード設定(T1)と合成して k 本目の**初回グレイン時刻 = spawn + 初回飛行**を assert(D13 の「発音は衝突のみ」の検証を兼ねる)。D11 減衰則のゲイン比(初速減衰なら初回衝突速度比 = `Bounce^(k−1)`)も閉形式オラクルで assert |
| T12 | Source アドレッシング | `head` は閉形式(§4.6): Motion=+100 で τ=PreDelay 一定(古典ディレイ遅延と一致)、Step=s で n 回目のグレインが `anchor0 + n·s` を読む(T1 の衝突時刻列と合成)、Spray=0 でビット決定性維持・Spray>0 で偏差 ±Spray 内、τ の write-head クランプと 34 s retire 境界を assert |

方針: 物理の検証は**回転や重力を切った可解ケース**に固定して閉形式オラクルと
突き合わせ、回転・偏心などの非可解領域は**不変量**(有界性・finite・決定性・
クランプ)で縛る。実装から期待値を導くテストは書かない。

## 14. 拡張ロードマップ(提言)

**v1.1(近い・v1 設計が受け皿を持つもの)**
- Sync Impact: シーケンス各ボールの**初回衝突**を `Time` グリッドに厳密一致させる
  (スポーン前に軌道を先読みして spawn 時刻を逆算 — RT 予算の検討が必要。
  D12/D13 の「呼吸」をあえて殺す opt-in のタイトモード)。
- Geo Scan: 接触点の箱上の角度→バッファ窓内の読み位置マッピング(回る箱が
  音声履歴を文字どおりスキャンする — Source クラスタ §4.6 の物理連動版)。
- `StepSync`: `Step` の音価シンク(ビート素材のフレーズスキャン用)。
- ボール衝突グレイン: `Ball Collide` on 時、ボール同士の衝突で両者のアンカーを
  ブレンドしたグレインを発火。
- 壁マテリアル: 辺ごとに Tone/Pitch オフセット(回る箱が音色シーケンサになる)。
- `Velocity→Level` 量ノブ(現在は固定マッピング §4.5)。
- 音声ループ Regen(要・不変量設計 = **人間相談**)。

**v2(中距離・新しい入出力面)**
- MIDI スポーン: note-on でボール射出、note→Pitch、velocity→Speed
  (「物理アルペジエータ・ディレイ」化)。
- Freeze: リング書き込み停止+ボール不死(無限グラニュラータンブラー)。
- 障害物: 箱内にペグ/バンパー配置(パチンコ・モード)。
- モジュレーションマトリクス: 物理観測量(衝突速度・位置・年齢・世代)→
  音響ターゲット(Pitch/Grain/Tone/Pan)。
- サイドチェインスポーン(別トラックのキックでボールを蹴る)。

**遠距離(コンセプト拡張)**
- マルチチェンバー+ポータル(箱 A から箱 B へボールが移動し音色が変わる)。
- 3D チェンバー(奥行き→リバーブセンド/幅)。
- 衝突→MIDI 出力(物理をトリガーシーケンサとして外部に提供)。
