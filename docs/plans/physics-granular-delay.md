## 0. 要約

入力のトランジェントを検出すると、回転する 2D ボックスの中にボールがスポーンする。
ボールは物理演算(初速・重力・ドラッグ・反発)で運動し、**壁に衝突するたびに、
そのボールを生んだ音のグレインが 1 発再生される**。つまり従来のディレイの
「ディレイタイム × フィードバック」を、「箱の幾何 × ボールの運動」が置き換える。
エコーのリズム・減衰・パンは物理から創発し、ビジュアライザはその物理そのもの。

- 従来のグラニュラーの `Density(Hz)` に相当するものは**存在しない** — 密度は
  箱のサイズ・回転・ボールの速度と数から創発する。
- パラメータの異なるボールを最大 4 スロット(A–D)同時に飛ばせる。
- フィードバックは音声ループではなく**世代スポーン(Refeed)**で実現する
  (構造的に発散不可能 — §4.8)。

## 1. コンセプトと位置づけ

### 1.1 エレベーターピッチ

「音を投げ込むと、回る箱の中でコロコロ跳ねて返ってくるディレイ」。
リズムはテンポグリッドではなく重力と幾何が作る。壁にぶつかった瞬間だけ音が出る、
というルールが 1 つあるだけで、accelerando(重力バウンス)、poly-rhythm(多角形)、
うねる周期(偏心回転)がすべて自然に出てくる。

### 1.2 既存 Tatsunular Delay(`granular-delay`)との違い

| | Tatsunular Delay | 本機 |
|---|---|---|
| グレインの発火 | クロック駆動(`Density Hz` + jitter) | **衝突イベント駆動** |
| エコーの減衰 | feedback ゲイン(音声ループ) | 物理(反発・ドラッグ・寿命)+ Refeed 世代 |
| リズムの性格 | 定常的な雲・テクスチャ | 拍動・バウンス・回転周期 |
| パン | ランダム spread | 衝突位置(物理)由来 |

役割が異なるため共存させる(置き換えではない)。グレイン再生の実装語彙
(Hann 窓・rate 読み・プール)は `core/GranularDelay.h` と揃える。

### 1.3 名称候補(**人間判断 — taste**)

slug は英語・記述的(既存: `granular-delay`, `shimmer-reverb`)。仮称として本書では

## 2. 用語とオブジェクトモデル

- **World(ワールド)** — 1 つの 2D 空間。**箱は 1 個で全スロット共有**(§6-D2)。
- **Box(箱)** — 回転する正多角形または円。壁がグレインのトリガー面。
- **Slot(スロット)** — ユーザー提案の「ボールインスタンス」。ボールのパラメータ
  一式を持つ。A–D の 4 基、A のみデフォルト有効。
- **Ball(ボール)** — スロットから生まれる実体。生成時に「アンカー」(自分を生んだ
  入力音のリングバッファ上の時刻)を 1 つ持つ。
- **Collision(衝突)** — ボールと壁の接触イベント。1 衝突 = 最大 1 グレイン。
- **Grain(グレイン)** — Hann 窓の短い再生断片。リングバッファのアンカー近傍を読む。
- **Generation(世代)** — Refeed で親ボールから子ボールが生まれた回数(§4.8)。

## 3. シグナルフロー

```
in L/R ──┬───────────────────────────── dry ────────────┐
         ├─ mono sum ─ OnsetDetector ─ spawn events     ├─ Mix ─ Out
         └─ stereo ring buffer (18 s)                   │
                   ↑ read (anchored, per-grain rate)    │
              GrainEngine ── grains ─ Tone LP ── wet ───┘
                   ↑ collision events (sample-accurate)
              PhysicsWorld: Box(shape/size/spin/pivot/gravity)
                            × Balls(slot A–D params)
```

音声ループは存在しない(純フィードフォワード)。Refeed は「音を書き戻す」の
ではなく「減衰した子ボールを生む」(§4.8)。

## 4. 動作仕様

### 4.1 オンセット検出とスポーン

- 検出はモノサム上の 1 系統(グローバル)。`EnvelopeFollower.h`(fast/slow ペア)
  ベースのオンセット検出。
- パラメータ: `Sense`(しきい値 dBFS)、`Retrig`(再トリガー禁止時間)。
- **絶対 floor −80 dBFS**(退行クラス J): floor 以下の入力では絶対にスポーン
  しない。silence テスト必須。
- 1 オンセット = 有効な全スロットに 1 イベント配信。各スロットは自分の
  `Pre-Delay` 経過後にボールを 1 個スポーンする。
- スロットの生存数が `Count` に達していたら**最古のボールを消して**生む
  (oldest-steal。§6-D1)。
- スポーン位置: 箱の中心 + `Spawn Spread`(半径 R に対する % の一様円板ジッタ)。
  完全対称な初期条件は退化軌道(同じ 2 点を往復)に落ちるため、デフォルトで
  少量のジッタを入れる。
- 初速の向き: スロットの `Direction`(角度)を `Dir Random` % で一様ランダムに
  ブレンド。デフォルトは 100%(完全ランダム)。

### 4.2 物理ワールド — 座標系・単位・時間スケール

- 単位系: 箱の外接円半径 **R = 1**。物理は秒領域で積分(サンプルレート非依存)。
- **`Box Size` が「時間スケール」を決める**(未明言だった最重要パラメータ、§6-D3):
  `Box Size = T` 秒 ⇔ 「基準速度のボールが箱の直径 2R を横断するのに T 秒かかる」。
  すなわち基準速度 `vRef = 2R / T`。ボールの実速度 = `Speed × vRef`。
  → 体感的に `Box Size` ≈ ディレイタイム、`Speed` ≈ その分周/逓倍として効く。
- `Box Size` はテンポシンク可(音価 = 横断時間)。
- ボール半径は R 比 %。`min(有効半径) ≤ 0.4 × 内接円半径` にクランプ
  (箱よりデカいボール対策)。

### 4.3 回転する箱 — 移動壁とエネルギー、クランプ

- `Spin`: 双極性(±方向)、free では rev/s、sync では「1 回転あたりの音価」。
  デフォルトは 0 ではなく**ゆっくり回っている**(ユーザー提案の「デフォルトで
  四角のボックスが回っている」を反映。初期値は taste — §15)。
- `Pivot X/Y`: 回転軸の位置。(0,0)=中心。±1 × R の範囲。軸をずらすと箱が
  偏心して振れ、衝突タイミングが周期的にうねる(ワウ的なグルーヴ)。
- 衝突は**移動壁**として解く: 接触点の壁速度 `v_wall`(回転による接線速度)を
  使い、相対速度の法線成分を反発させる
  `v' = v − (1 + e) · ((v − v_wall)·n) n`。
  → **回転はボールにエネルギーを注入できる**(タンブラーが中身をかき回す)。
  これは意図した仕様(回転を上げるとエコーが死なずに暴れ続ける)。
- 安全クランプ(必須): ボール速度上限 `|v| ≤ 8 × vRef`。グレインゲインは
  §4.5 のマッピングで **1.0 で飽和**。→ 長時間ホールドでも出力は有界
  (退行クラス C の「現実的ピーク上限」テスト対象)。
- `Ball Collide` トグル(デフォルト off): ボール同士の弾性衝突(質量 ∝ r²)。
  on でも**ボール同士の衝突はグレインを発火しない**(v1。拡張 §14)。
  壁衝突のみが `Life Bounces` にカウントされる。

### 4.4 ボールの運動 — 「加速度(減衰度)カーブ」の分解

ユーザー提案の「初速、加速度(減衰度)カーブ」を次の 4 つに分解する(§6-D4):

| 要素 | パラメータ | 物理 | 音への効き |
|---|---|---|---|
| 初速 | `Speed`(スロット) | スポーン時の速さ | エコー密度の基準 |
| 定加速度 | `Gravity`(グローバル) | 一定の下向き加速度 | バウンスの accelerando。0 = ビリヤード(等速) |
| 経時減衰 | `Drag`(スロット) | 速度比例の空気抵抗 | エコー間隔が徐々に開く(rit.) |
| 減衰の形 | `Decay Curve`(スロット) | Drag 係数をボール年齢で変化させるプロファイル(前重心 ↔ 線形 ↔ 後重心) | 「最初に一気に落ちる」↔「最後に急に死ぬ」 |
| 衝突減衰 | `Bounce`(スロット、反発係数 e) | 衝突ごとのエネルギー保持率 | 1 衝突ごとの音量減衰 = feedback 相当 |

- 重力は**ワールド(画面下)固定**で箱と一緒に回らない。回る箱 × 固定重力 =
  タンブラーの気持ちよさの源泉。
- e(`Bounce`)の上限は **100%**(>100% の増幅反発は禁止 — 退行クラス A 域。
  拡張候補 §14)。
- 静止判定: 法線衝突速度 `< 0.02 × vRef` の接触はグレインを発火せず滑らせる
  (転がり接触のマシンガン化防止)。加えてボールごとにグレイン発火の最小間隔
  15 ms を内部固定で持つ。

### 4.5 衝突 → グレイン変換

- **タイミング**: 衝突時刻はサンプル精度でグレイン開始に反映(§4.10)。
- **ゲイン**: `g = min(1, |v_impact·n| / vRef)`。反発 e のボールは衝突ごとに
  インパクト速度が e 倍 → グレインゲインも e 倍(検証しやすい独立オラクル、
  §10-T2)。上限 1.0 で飽和(壁で音が増えることはない)。
- **パン**(`Pan Mode`、スロット):
  - `Physics`(デフォルト): 接触点の x 座標(ワールド系)→ 等パワーパン。
    箱の左壁に当たれば左から鳴る。ビジュアライザと音像が一致する。
  - `Center` / `Random`: 従来型。
- **グレイン内容**: リングバッファをアンカー位置(§4.6)から `Grain` 長ぶん、
  Hann 窓、per-grain 再生 rate = `2^((Pitch + PitchRand·rand)/12)`。
  `Reverse` % の確率で逆再生。実装語彙は `GranularDelay.h` の Grain 構造を踏襲
  (プール固定・rate 読み・d クランプ)。
- グレインプールはグローバル 64(core 精例 `kMaxGrains` と同値)。満杯時は
  新規をドロップ(core と同じポリシー)。

### 4.6 グレインソースのアンカリングと `Warp`(§6-D5)

ボールは生成時に「自分を生んだオンセットのリングバッファ時刻」= **アンカー**を
記憶する。衝突時のグレイン読み出し位置:

```
readPos(collision) = anchor + Warp × (t_collision − t_spawn)
```

- `Warp = 0%`(デフォルト): 全ての衝突が**誕生時のトランジェント**を再生する。
  「ボール = その音の化身」というコンセプトの核。スタッター/リズミック。
- `Warp = 100%`: 読み位置が実時間で進む = 衝突時刻の `Pre-Delay` 前の音を再生
  = 古典的なディレイの挙動(入力に追従)。
- 中間値はその補間(テープの引き伸ばし的な遅れ方)。

従来グラニュラーの `Position Jitter` はこの機構(+物理由来のタイミング揺れ)が
置き換えるため、独立パラメータとしては持たない。

### 4.7 寿命と消滅

- `Life Mode` トグル(ユーザー提案どおり):
  - `Time`: 0.1–16 s。経過で消滅。
  - `Bounces`: 1–99 回。**壁反射のみ**カウント。
- 消滅時: 進行中のグレインは最後まで再生(新規発火だけ止める)。ビジュアルは
  フェードアウト。クリックは構造上出ない(グレインは常に窓済み)。
- 暗黙の消滅(モードに関係なく常時):
  1. エネルギー死: `|v| < 0.01 × vRef` が 250 ms 続いたら消滅(ゾンビ防止)。
  2. ホライズン死: アンカーがリングバッファの保持限界を超えたら消滅(§4.9)。

### 4.8 Refeed — 世代フィードバック(音声ループを持たない feedback)

未明言だった「フィードバック」への回答(§6-D6)。**音声ループ方式は採らない**:
衝突駆動ではグレインの重なりが不定で、N 個のボールが同一アンカーを同時に読むと
ループゲインが `feedback × N` になり得る(退行クラス A の設計違反)。代わりに:

- `Refeed`(グローバル、0–95%): ボールが**消滅するとき**、確率/ゲイン `Refeed`
  で**子ボール**を最終衝突位置からスポーンする。
- 子は親のパラメータ一式を継承しつつ:
  - `gainScale(child) = gainScale(parent) × Refeed` — 世代ごとに幾何級数で減衰。
  - `Pitch` はスロット値を**累積**(gen 2 = 2×Pitch semitones …)。
    → 音声ループなしでシマー的なピッチ螺旋が得られる。
  - アンカーは親のものを継承(同じ音の残響世代)。
- 世代上限 8。`gainScale < −60 dB` でも打ち切り。
- **構造的に発散不可能**(純フィードフォワード、各世代ゲイン ≤ Refeed^gen < 1)。
  クラス A のワーストケーステストは「それでも」全レートで回す(§10-T3)。

古典的な音声ループ Regen(ウェットをリングに書き戻す)は v1 非搭載。搭載する
場合はループゲイン正規化の不変量設計が必要(拡張 §14、人間相談案件)。

### 4.9 メモリホライズン(§6-D7)

`Warp < 100%` のグレインは過去のアンカーを読み続けるため、リング長が寿命を規定する:

- リングバッファ = **18 s**(= max Life Time 16 s + max Pre-Delay 1 s +
  max Grain 0.5 s + rate マージン)。ステレオ。`prepareToPlay` で最悪ケース確保
  (退行クラス D)。
- メモリ見積: 48 kHz float ≈ 6.9 MB、192 kHz float ≈ 27.6 MB
  (double なら ×2 — 格納精度は実装時に既存 `DelayLine` の精度に合わせて決定)。
- アンカー年齢が 18 s を超える状況(Bounces モードの超スローボール等)では
  ボールをホライズン死させる(クランプで誤った音を読ませない — クラス D の
  「明示クランプ+テスト」)。

### 4.10 決定性とサンプルレート非依存(§6-D8)

- 物理は**固定 1 kHz ティック**(秒領域、絶対時刻アンカー)で積分し、衝突を
  検出したティック内で**二分法(上限 8 反復)でサンプル精度に精密化**する。
  → タイミングはサンプルレートにもブロックサイズにも依存しない。
- 乱数は core 精例の xorshift。シードはボールごとに決定的
  (`hash(slotIndex, spawnCounter)`)。→ 同一入力・同一パラメータなら
  オフラインレンダーは**ビット再現**(§10-T6 で 64 vs 512 サンプルブロックの
  一致を検証)。
- 1 ティック内の多重衝突は TOI 順に逐次解決、ボールあたり上限 4 回/ティック
  (RT 上限)。ボールが壁にめり込む遷移(箱の形状/サイズ/軸の変更中)は法線方向
  push-out で解消し、速度は増やさない(爆発防止)。

| `Grain` | Grain | 10–500 ms(log) | 90 ms | core の kMaxGrainMs=500 と整合 |
| `Reverse` | Reverse | 0–100 % | 0 % | 逆再生確率 |
| `Warp` | Warp | 0–100 % | 0 % | アンカー追従(§4.6) |
| `Gain` | Slot Gain | −24–+6 dB | 0 dB | |

計 94 パラメータ。全スロットぶんを常時 APVTS に宣言(off スロットは UI 側で
非活性表示)。UI が窮屈なら `DirRandom`/`PitchRand` は折りたたみ候補。

### 5.3 従来グラニュラーディレイのパラメータとの対応

「基本的なグラニュラーディレイのパラメータ」(ユーザー提案)の消化先:

| 従来 | 本機での対応 |
|---|---|
| Delay Time | `Box Size`(+ `Speed`, `Pre-Delay`) — 幾何から創発 |
| Feedback | `Bounce`(衝突減衰)+ `Life` + `Refeed`(世代) |
| Density (Hz) | **なし** — 衝突レートから創発 |
| Grain Size | `Grain`(そのまま) |
| Pitch / Pitch Rand | `Pitch` / `PitchRand`(そのまま) |
| Position Jitter | **なし** — 物理タイミング + `Warp` が置換 |
| Spread(パン) | `Pan Mode`(Physics/Center/Random) |
| Mix / Tone | `Mix` / `Tone`(そのまま) |

### 5.4 平滑化と適用タイミング(退行クラス F)

- 毎サンプル `SmoothedValue`: `mix` `outGain` `tone` `Gain`(音声経路の連続量)。
- 物理パラメータ(`spin` `boxSize` `pivot` `gravity` 等)は 1 kHz ティックで
  スムージング適用(イベント列にしか影響しないためジッパーは構造上出ない。
  `boxSize` 急変は §4.10 の push-out で安全)。
- グレイン単位でラッチ: `Pitch` `Grain` `Reverse` `Warp`(発火時に確定、
  再生中は不変 — クリックなし)。

## 7. UI 仕様(factory_ui 準拠)

- レイアウト:
  - ヘッダ: FactoryChrome + `PresetSelectorController`(標準)。
  - 左 ~55%: **ビジュアライザ**。回転する箱(無信号でも回る — ユーザー提案の
    既定状態)、スロット色のボール、衝突フラッシュ+リップル、短い残像。
    `Pivot` は箱上のハンドルを直接ドラッグして編集(対応 APVTS パラメータに
    ジェスチャ通知)。HUD: 推定初回エコー ms / スロット別生存数。
  - 右カラム: World(Shape/Size/Spin/Pivot/Gravity/Ball Collide)、
    Detect(Sense/Retrig/Spread)、Out(Refeed/Tone/Mix/Output)。
  - 下段: スロットタブ A–D(有効 LED + 選択)+ 選択スロットのノブバンク。
- 色: `factory_ui` パレットからスロット別アクセント 4 色(実装時に `factory-ui`
  スキルで確認。パレットに 4 アクセントが無ければ**人間相談** — パレットの
  fork は禁止されているため)。
- 数値表示は共有ヘルパ(`setSliderDecimals` 等)で桁固定(退行クラス K)。
- **スレッド契約**(退行クラス I): 物理は**オーディオスレッドの DSP**(タイミング
  が音楽的本体のため。UI スレッド物理は不可)。UI へは
  - イベント SPSC FIFO(spawn/hit/die、有界・上書き)
  - 60 Hz 位置スナップショット(atomic インデックスのダブルバッファ)
  で一方向ミラーする。エディタ閉時は書き捨て。UI→DSP は APVTS 経由のみ。

## 9. RT 安全と退行防止ゲート(クラス対応)

`processBlock` 系は allocation/lock/syscall なし(プール・リング・FIFO は
`prepareToPlay` で最悪ケース確保)。pluginval strictness 5 対象。

| クラス | 本機での対応 |
|---|---|
| A 発散 | 音声ループ自体を持たない(Refeed は世代ゲイン ≤ Refeed^gen)。それでも最悪ケース(最大 Spin/Bounce/Refeed/32 балls)でインパルス応答エネルギー非増加を全 6 レート検証 |
| B 共振上限 | 共振フィルタなし(Tone は 1-pole)。該当ゲートなしを明記 |
| C finite | グレイン合算→出力ノードに finite ガード(非 finite でグレイン kill + リングflush、自己回復)。長時間ホールドで `allFinite` + **peak < 4.0**(現実的上限) |
| D バッファ | リング 18 s を最悪ケースから導出(§4.9)。ホライズンは明示クランプ(ボール retire)+テスト |
| E リセット | `prepareToPlay` 末尾で `reset()`(全ボール kill・リング/検出器/RNG 初期化)。bypass 解除・channel 遷移でも同様 |
| F 平滑化 | §5.4 のとおり(SmoothedValue / ティック平滑 / グレインラッチ) |
| G レート行列 | 全テスト `sampleRatesFromArgs()`。テスト内 FFT は `fftOrderForSampleRate` |
| H バグ経路 | ワースト設定(最大 Spin・e=1・Drag 0・全ボール)を実際に踏む。オラクルは §10(解析解) |
| I atomic | ビジュアライザ共有は SPSC FIFO + atomic ダブルバッファのみ(§7) |
| J 絶対 floor | 検出器 −80 dBFS floor + silence テスト(スポーン 0・ウェット 0) |
| K GUI | factory_ui 共有ヘルパで桁・幅・描画空間を統一 |
| L コム | dry と wet は「別時刻の音」の混合であり並列同相合成ではない(クラス L 非該当を明記)。equal-gain mix |
| M 線形位相 | FIR クロスオーバー不使用(非該当) |

## 14. 拡張ロードマップ(提言)

**v1.1(近い・v1 設計が受け皿を持つもの)**
- `Burst`: 1 オンセットで Count 個を同時射出(D1 の別解釈をトグルで回収)。
- ボール衝突グレイン: `Ball Collide` on 時、ボール同士の衝突で両者のアンカーを
  ブレンドしたグレインを発火。
- 壁マテリアル: 辺ごとに Tone/Pitch オフセット(回る箱が音色シーケンサになる)。
- `Velocity→Level` 量ノブ(現在は固定マッピング §4.5)。
- 音声ループ Regen(要・不変量設計 = **人間相談**)。

**v2(中距離・新しい入出力面)**
- MIDI スポーン: note-on でボール射出、note→Pitch、velocity→Speed
  (「物理アルペジエータ・ディレイ」化)。
- Freeze: リング書き込み停止+ボール不死(無限グラニュラータンブラー)。
- 障害物: 箱内にペグ/バンパー配置(パチンコ・モード)。
- モジュレーションマトリクス: 物理観測量(衝突速度・位置・年齢・世代)→
  音響ターゲット(Pitch/Grain/Tone/Pan)。
- サイドチェインスポーン(別トラックのキックでボールを蹴る)。

**遠距離(コンセプト拡張)**
- マルチチェンバー+ポータル(箱 A から箱 B へボールが移動し音色が変わる)。
- 3D チェンバー(奥行き→リバーブセンド/幅)。
- 衝突→MIDI 出力(物理をトリガーシーケンサとして外部に提供)。
